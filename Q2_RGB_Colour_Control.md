While building circuit, I have taken Arduino connected to the breadboard. RGB LED's cathode is grounded where the other are connected to the arduino digital PWM~ pins with resistor of 220 Ohms. And an LED is taken and its cathode is grounded and anode is connected to arduino pin with resistor of 220 Ohms. A potentiometer is taken and its VCC is connected to 5V, GND is grounded and wiper is connected to analog pin A0.
In this code I have taken help of millis function. The potentiometer give the sensor values from 0 to 1023. So for some better result I have taken a variable val is is one fourth of sensor value of the potentiometer.
As per my code Red increases with sensor value because it is directly set to val. Green decreases because I have set it to 255-val which is inverse relation. Similar to red, Blue increases but slowly because it was set to half of val.
The LED also blinks up as up the potentiometer values keeps changing. A variable called interval which is add a minimum delay to prevent fast blinking checks the time difference between current running time to previously happend action relationship. Each time this sequence satisfies the LED change its state from ON/OFF.
